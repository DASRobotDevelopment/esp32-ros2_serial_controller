# ROS2-ESP32 Serial Controller

Этот проект реализует простой текстовый протокол по UART между ROS2‑узлом (или любым ПК) и контроллером на базе ESP32 для управления четырёхколёсным роботом с PID‑регулированием скорости.

## Возможности

- Управление четырьмя DC‑моторами с энкодерами.
- Задание ШИМ (PWM) напрямую или по целевой скорости в RPM.
- PID‑контроль скорости для каждого колеса.
- Настройка PID‑параметров «на лету» через Serial.
- Передача на хост текущих значений энкодеров и RPM.
- Отладочная информация для диагностики работы контроллера.

## Аппаратные параметры

Основные параметры задаются в коде:

- `NUMBER_OF_WHEELS` — количество колёс (мотор‑энкодерных пар).
- `MOTOR_PPR` — количество импульсов энкодера на один оборот.
- `MOTOR_MAX_RPM` — максимальные обороты мотора.
- `PID_UPDATE_FREQ` — частота обновления PID‑регулятора (Гц).

Пины моторов и энкодеров задаются в `Pins.h` через массивы `motorPins` и `encoderPins`.

## Последовательный протокол

Все команды отправляются в виде строк ASCII, заканчивающихся `\n` или `\r\n`.

### Команды

- `e`  
  Запрос значений энкодеров.  
  Ответ:  
  `e,<enc0>,<enc1>,<enc2>,<enc3>`

- `m,<pwm0>,<pwm1>,<pwm2>,<pwm3>`  
  Прямая установка PWM для каждого мотора (обход PID).  
  Значения `pwmN` — целые числа (обычно в диапазоне, поддерживаемом классом `Motor`).

- `v,<rpm0>,<rpm1>,<rpm2>,<rpm3>`  
  Установка целевых скоростей вращения (RPM) для PID‑контроллера каждого колеса.  
  Значения `rpmN` — числа с плавающей точкой.

- `p,<kp>,<ki>,<kd>`  
  Установка коэффициентов PID‑регулятора (одинаковых для всех моторов).  
  Параметры — числа с плавающей точкой.

- `s`  
  Запрос статуса.  
  Ответ:  
  `s,<enc0>,<rpm0>,<enc1>,<rpm1>,<enc2>,<rpm2>,<enc3>,<rpm3>`

- `t`  
  Запрос отладочной/тестовой информации по каждому PID‑контроллеру  
  (формат определяется реализацией `PIDController::showTestInfo()`).

## Архитектура кода

- `Motor` — управление направлением и ШИМ мотора.
- `MotorEncoder` — подсчёт импульсов энкодера и предоставление значения счётчика.
- `PIDController` — расчёт управляющего воздействия (PWM) по целевому RPM и данным энкодера.
- Главный цикл:
  - периодически вызывает `computePWM()` для всех PID‑контроллеров;
  - проверяет входящие данные по Serial и обрабатывает команды.

Парсинг параметров команд реализован так, чтобы легко масштабироваться под любое значение `NUMBER_OF_WHEELS` без изменения форматных строк — используется разбор списков значений, разделённых запятыми.

## Подключение к ESP32

### Распиновка колес

**Переднее левое колесо** подключаем к пинам:  
- Энкодер A: **GPIO 32**  
- Энкодер B: **GPIO 33**  
- Мотор PWM1: **GPIO 14**  
- Мотор PWM2: **GPIO 27**

**Переднее правое колесо** подключаем к пинам:  
- Энкодер A: **GPIO 5**  
- Энкодер B: **GPIO 4**  
- Мотор PWM1: **GPIO 21**  
- Мотор PWM2: **GPIO 22**

**Заднее левое колесо** подключаем к пинам:  
- Энкодер A: **GPIO 26**  
- Энкодер B: **GPIO 25**  
- Мотор PWM1: **GPIO 12**  
- Мотор PWM2: **GPIO 13**

**Заднее правое колесо** подключаем к пинам:  
- Энкодер A: **GPIO 18**  
- Энкодер B: **GPIO 19**  
- Мотор PWM1: **GPIO 16**  
- Мотор PWM2: **GPIO 17**

### Pins.h — автоматическая конфигурация

Файл `Pins.h` содержит готовые массивы для индексации:

```cpp
int encoderPins[4][2] = {
    {32, 33},  // переднее левое
    {5,  4},   // переднее правое  
    {26, 25},  // заднее левое
    {18, 19}   // заднее правое
};

int motorPins[4][2] = {
    {14, 27},  // переднее левое
    {21, 22},  // переднее правое
    {12, 13},  // заднее левое  
    {16, 17}   // заднее правое
};
```

**Подключение готово** — код автоматически найдёт все колёса по индексам 0-3!

## Использование

1. Залейте прошивку на ESP32.
2. Подключите ESP32 к ROS2‑узлу или ПК (USB/UART).
3. Настройте скорость порта (по умолчанию 115200 бод).
4. Отправляйте команды в описанном формате и обрабатывайте ответы на стороне ROS2/ПК.
